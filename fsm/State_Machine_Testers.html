procedure

(sm-test-invs m 
  [#:rep-limit rep-limit 
  #:max-path-length path-length] 
  #:ds-remove ds-remove 
  invs ...) 
 → (listof (list state word))
  m : machine?
  rep-limit : natural? = 1
  path-length : natural? = 12
  ds-remove : #t
  invs : (list state (-> word boolean))
Tests the given machine using the given state invariants. Dead states are removed for testing purposes, ds-remove is used to determine if they are removed. If m is a finite-state machine, then rep-limit is used to limit the number of times a transition is used. If m is a pda, then max-path-length is used to limit the number of steps a computation can take. If all tests pass, then the function returns the empty list. Otherwise, the function returns a list of pairs, where each pair contains a word and the state for which the invariant failed.

Note: Currently does not work with Turing machines, and mttms.

procedure

(sm-quickcheck m    
  [#:num-tests tests    
  #:max-length-kleenes num-kleenes]    
  invs ...) → void?
  m : machine?
  tests : natural? = 300
  num-kleenes : natural? = 20
  invs : (list state (-> word boolean))
Tests the given machine using the given state invariants. Dead states are removed for testing purposes. ds-remove is used to determine if they are removed. tests is the number of tests to perform on each state. num-kleenes is the maximum number of times a loop is traversed in the given machine’s transition relation. If tests pass, the function returns void. If tests fail, failed test reports are generated.

Note: Currently does not work with pdas, Turing machines, and mttms.

procedure

(sm-test m1) → (listof (list word symbol))

  m1 : machine?
(sm-test m1 n) → (listof (list word symbol))
  m1 : machine?
  n : natural?
Applies the given machine to randomly generated words and returns a list of words and the obtained result. If the given machine is a Turing machine, it must be a language recognizer. For a Turing machine language recognizer, the generated tests start with the left-end marker followed by the input word and the head on the first letter of the input word. The optional natural number specifies the number of tests (the default is 100). May not be used for Turing machines and multitape Turing machines.

procedure

(sm-sameresult? m1 m2 w) → boolean

  m1 : machine?
  m2 : machine?
  w : word
Tests if the two given machines return the same result when applied to the given word.

procedure

(sm-testequiv? m1 m2 n) → (or boolean (listof word))

  m1 : machine?
  m2 : machine?
  n : natural?
Tests if the two given machines return the same result when applied to the same 100 randomly generated words. Returns true if all results are the same. Otherwise, a list of words for which different results were obtained is returned. May not be used for Turing machines and multitape Turing machines.
