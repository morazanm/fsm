3 State Machine Constructors

procedure

(make-dfa sts sigma start finals delta) → dfa

  sts : (listof state)
  sigma : alphabet
  start : state
  finals : (listof state)
  delta : (listof dfa-rule)
Builds a deterministic finite-state automaton. delta is a transition function.

procedure

(make-ndfa sts sigma start finals delta) → ndfa

  sts : (listof state)
  sigma : alphabet
  start : state
  finals : (listof state)
  delta : (listof ndfa-rule)
Builds a nondeterministic finite-state automaton. delta is a transition relation.

procedure

(make-ndpda sts sigma gamma start finals delta) → ndpda

  sts : (listof state)
  sigma : alphabet
  gamma : (listof symbol)
  start : state
  finals : (listof state)
  delta : (listof pda-rule)
Builds a nondeterministic pushdown automaton from the given list of states, alphabet, list of stack symbols, start state, list of final states, and list of pda-rule. delta is a transition relation.

procedure

(make-tm sts sigma delta start finals) → tm

  sts : (listof state)
  sigma : alphabet
  delta : (listof tm-rule)
  start : state
  finals : (listof state)
(make-tm sts sigma delta start finals accept) → tm
  sts : (listof state)
  sigma : alphabet
  delta : (listof tm-rule)
  start : state
  finals : (listof state)
  accept : state
Builds a nondeterministic Turing machine. delta is a transition relation. LM is automatically added to the machine’s alphabet. Rules for moving off the LM are automatically added to the machine’s rules. If the optional accept argument is given then the resulting Turing machine is a language recognizer.

Builds a deterministic finite-state automaton equivalent to the given ndfa.

procedure

(regexp->fsa r) → ndfa

  r : regexp
Builds a fsm for the language of the given regular expression.

procedure

(sm-rename-states sts m1) → state-machine

  sts : (listof state)
  m1 : state-machine
Builds a state machine that is excatly the same as the given machine except that its states are renamed as to not have a name in common with the given list of states.

procedure

(sm-union m1 m2) → state-machine

  m1 : state-machine
  m2 : state-machine
Builds a state machine for the language obtained from the union of the languages of the two given state machines. If the inputs are Turing machines then they must be language recognizers. The given machines must have the same type. May not be used with multitape Turing machines.

procedure

(sm-concat m1 m2) → state-machine

  m1 : state-machine
  m2 : state-machine
Builds a state machine for the language obtained from the concatenation of the languages of the two given state machines. [Note: Not yet implemented for Turing machines language recognizers. May not be used with multitape Turing machines.].

procedure

(sm-kleenestar m1) → state-machine

  m1 : state-machine
Builds a state machine for the language obtained from the Kleene star of the given machine’s language. May not be used with multitape Turing machines nor with Turing machine language recognizers.

procedure

(sm-complement m1) → state-machine

  m1 : state-machine
Builds a state machine for the language obtained from the complement of the given machine’s language. The given machine can not be a ndpda. If the inputs are Turing machines then they must be language recognizers. May not be used with multitape Turing machines.

procedure

(sm-intersection m1 m2) → state-machine

  m1 : state-machine
  m2 : state-machine
Builds a state machine for the language obtained from the intersection of the languages of the two given state machines. If the inputs are Turing machines then they must be language recognizers. The given machines must have the same type. May not be used with multitape Turing machines.

procedure

(grammar->sm g) → state-machine

  g : grammar
Builds a state machine for the language of the given regular or context-free grammar.
